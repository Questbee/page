Your team is heading to the field next week. Someone found Questbee — a free survey tool that works without internet once it's set up — and now you're the one who has to get it running. You've looked at the homepage, it seems like exactly what you need, and you're wondering how hard this is going to be.

Not hard at all. The whole setup takes about 30 minutes on a laptop or a small server you already have. You'll answer a few questions in a guided wizard, and by the end of it your team will have a working platform with a web dashboard and a mobile app ready to go into the field.

This guide walks through every step. If you get stuck, the full documentation is always there — but most people don't need it.

Before you start

Questbee runs on your own computer or server — that's what makes it free and what keeps your data under your control. Before the setup wizard can do its work, three tools need to be installed on that machine: Docker, Git, and Python 3.

Think of Docker as the engine that runs everything. Git is how you get the Questbee files. Python is what runs the setup wizard. None of them require any configuration on your part — just installation.

ℹ️

Never installed these before? The Prerequisites guide has step-by-step instructions for Windows, macOS, and Linux — including what to click and where to download each one.

To check whether you already have them, open a terminal window and run these four lines one by one:

docker --version
docker compose version
git --version
python3 --version

If each one prints a version number, you're ready. If any of them shows an error, the prerequisites guide will sort you out in a few minutes.

You'll also want the machine to have at least 4 GB of memory and 50 GB of free disk space — roughly what a modern laptop has by default.

Get the files

Open a terminal and navigate to the folder where you want to install Questbee. Then run:

git clone https://github.com/Questbee/community.git
cd community

This downloads the Questbee files to your machine and moves you into the project folder. Everything from here happens inside that folder.

Run the setup wizard

This is the part where it gets easy. One command starts a guided wizard that asks you a handful of questions and handles the rest:

./questbee install

The wizard will ask you whether you're setting this up for local use (just your machine) or for a live server, then set a password for the database and generate a security key. You don't need to write any of this down — it's saved automatically in a configuration file.

✅

When the wizard finishes, everything is already running. You don't need to start anything manually or edit any files.

Once it's done, confirm that all three parts of the system are up:

./questbee status

You should see the web dashboard, the API, and the database all listed as running. If something isn't running, the wizard output usually tells you why.

Log in for the first time

Open a browser and go to http://localhost:3000 — or, if you installed on a separate server, swap localhost for that machine's IP address on your network.

ℹ️

The wizard printed the exact address at the end. If you missed it, run ./questbee hostname to see it again.

On the first visit, Questbee creates an admin account automatically. The username and password are shown at the end of the wizard output. Copy them before closing that terminal window.

Log in and you'll land on the main dashboard. That's your home base — from here you manage forms, review what your team submitted, and control which devices are connected. Take a minute to look around before moving on.

Create your first form

Click Forms in the left sidebar. This is where all your data collection forms live. Click New Form to open the form builder.

1 Name your form

Give it a name that your field team will recognize — something like "Household Visit — May 2025" or "Water Point Inspection."

2 Add your fields

Drag in the fields you need. Questbee supports text answers, numbers, dates, dropdown choices, GPS location, photo capture, and more. Build it the same way you'd design a paper form — section by section, question by question.

3 Publish it

When the form looks right, hit Publish. This is the step most people miss the first time around.

⚠️

A form that isn't published won't appear on your team's phones. Save and publish before you hand out devices.

Connect the mobile app

The Questbee mobile app is where your field team does their work. It runs on Android, stores everything on the phone, and syncs back to your dashboard when there's a connection — even a brief one.

Connecting a phone takes about two minutes and uses a QR code, so your team doesn't have to type anything.

1 Find your server address

In the terminal, run:

./questbee hostname

This shows the local network address of your machine — something like 192.168.1.100. Copy it. This is what the phone will use to find your server.

2 Open the Mobile App section

In the web dashboard, click Mobile App in the sidebar. Paste the address you just copied into the Server Address field. The dashboard shows you a preview of the full address it will use — check that it looks right before continuing.

3 Name the device and generate a QR code

Enter a name for this device — "Field Tablet 1" or the name of the person who will carry it. Then click Generate QR Code.

4 Scan on the phone

Open the Questbee app, tap Pair with QR Code, and point the camera at the screen. The phone connects automatically and downloads all your published forms. It's ready to go into the field.

✅

Setting up a whole team? Click Pair another device after each scan — the server address stays saved and you just generate a new code for each phone.

Keeping things running

Once Questbee is set up, managing it is straightforward. A small set of commands covers everything you'll need day to day:

  • ./questbee start — start everything (use this after a reboot)
  • ./questbee stop — shut everything down cleanly
  • ./questbee status — check what's running
  • ./questbee logs — see what's happening in real time
  • ./questbee update — pull the latest version of Questbee
  • ./questbee hostname — remind yourself of your server's address

That's the whole system. Your forms are ready, your phones are paired, and your team can start collecting data — with or without internet. When they get back and connect to the network, everything syncs automatically and appears in your dashboard.

If you want to go further — set up a public domain, configure user roles, or explore the full feature set — the Questbee documentation covers all of it. Or just start with what you have and build from there.
